## POST /auth/register

Register a user

### Request

#### Body

-   `email` (string, required): The email of the user
-   `password` (string, required): The password of the user
-   `name` (string, required): The name of the user
-   `frontendUrl` (string, required): The frontend url of the verify page

:::danger

The user will receive an email to verify the account

In the email a link goes to `frontendUrl`?q=token

Where you have to call the `GET /auth/verify` endpoint with the verify token in the q parameter

:::

### Response

<Tabs>
  	<TabItem value="200" label="200" default>
        User registered successfully
    </TabItem>
    <TabItem value="400" label="400">
        Bad request

        You can show the message in the response on the frontend to the user

        ```js
        {
            "status": "error",
            "message": "This email is already in use"
        }
        ```
    </TabItem>
    <TabItem value="422" label="422">
        Unprocessable entity

        ```js
        {
            "status": "error",
            "message": "One or more fields are missing"
        }
        ```
    </TabItem>

</Tabs>
