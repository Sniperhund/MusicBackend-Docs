## POST /user/password

Change the password of a user

:::info

You have to attach a access token in the Authorization in the header

:::

### Request

#### Body

-   `oldPassword` (string, required): The old password of the user
-   `newPWassword` (string, required): The new password of the user

### Response

<Tabs>
  	<TabItem value="200" label="200" default>
        Password changed successfully
    </TabItem>
    <TabItem value="400" label="400">
        Bad request

        You can show the message in the response on the frontend to the user

        ```js
        {
            "status": "error",
            "message": "Wrong password"
        }
        ```
    </TabItem>
    <TabItem value="401" label="401">
        Bad or expired token. The user needs to be authenticated.
    </TabItem>
    <TabItem value="422" label="422">
        Unprocessable entity

        ```js
        {
            "status": "error",
            "message": "One or more fields are missing"
        }
        ```
    </TabItem>

</Tabs>
